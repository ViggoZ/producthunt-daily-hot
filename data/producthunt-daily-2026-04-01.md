1. **使用 .git/info/exclude**  
   你可以在 `.git/info/exclude` 文件中添加需要排除的文件或文件夹。这是一个本地的排除列表，不会影响到其他开发者的 .gitignore 文件。

2. **使用 git update-index**  
   如果你已经将某个文件加入了 Git 的跟踪，你可以使用以下命令将其从跟踪中移除，但保留在本地：
   ```
   git update-index --assume-unchanged <file>
   ```
   这样，Git 会假装这个文件没有变化，不再跟踪它。

3. **使用 git stash**  
   如果你不想在本地保存某些文件的更改，可以在提交之前使用 `git stash` 将它们临时保存：
   ```
   git stash push <file>
   ```
   这样，你可以在提交后再应用这些更改。

通过这些方法，你可以灵活地管理本地文件，而无需修改 .gitignore 文件。
